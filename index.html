Project Idea: Alexa-Pi Robot

Overview

The project is focused on the development of a robot that is controlled via Amazon’s voice-assistant, Alexa. The aim of this project is to use the Amazon Alexa services to send voice commands to a robot to make it move around a room. This robot will consist of a Raspberry Pi to act as the communication relay and controller for the robot, and an Arduino Uno to control the motors of the robot. The user will then provide a voice command to the voice-assistant, which will be transmitted to and translated by the Raspberry Pi, before the command is sent to the Arduino Uno to perform the specific command. A Raspberry Pi camera is also attached that will be used to forward a camera feed or take pictures and display them on an Amazon Echo device.

Motivation

The development of this project will be interesting as it will provide a robot that can be driven using specific voice commands. There are also additional levels of complexity, as this project involves network configuration, as well as configuring serial communication between the Raspberry Pi and Arduino. There have been projects that have focused on developing robots that are controlled with Amazon’s Alexa services, and projects to use the Raspberry Pi camera as a security camera that can be displayed on Amazon’s Echo devices. As such, combining these two concepts together would result in a smart mobile security camera that could move around a specific area.

Description

The robot that is to be designed will have several different features incorporated into its design. This robot will be interfaced with Amazon Alexa, to allow the user to command the robot with their voice. Using Amazon Alexa, the user will be able to provide simple commands such as to move or rotate a certain distance. The user will also be able to request Alexa to display a live video or still image from the camera that is built into the robot. For example, the user might use the following phrase:

Alexa, turn the robot 30° clockwise.

Alexa, show me the view from the robot.

The above commands would cause the robot to rotate 30° clockwise from its current position. The next command would access the onboard camera and transmit the live feed to the screen of the Echo device through a secure TLS connection. To be able to view the video stream from the robot, an Amazon Echo Show or Amazon Echo Spot is required. Other commands that can be given will include requesting the robot to move in a certain direction or taking an image.

The robot will be constructed with a light-weight chassis made of MDF board. The chassis will have two stories, to increase the compactness of the robot. The Arduino and the motorised wheels will be placed on the bottom layer, while the Raspberry Pi and camera are placed on the top layer. The motors will be placed on either side of the chassis, with a caster while placed facing forward to support and balance the robot. Raspberry Pi camera will be placed facing forward to capture everything in the direction that is in the robot’s field of view.

The robot will make use of a Raspberry Pi to act as the main processor. All commands from Alexa will be given and processed through the Raspberry Pi. The in-built Raspberry Pi WiFi module will provide the robot with the ability to receive commands from the Alexa voice-assistant. The Raspberry Pi will also be connected to a camera which will be used to take photos or display a live feed that can be requested by Alexa to display to the user. To control the movement and direction of the robot, an Arduino Uno will be used for its ability to read and write analogue signals. This will increase the precision of the movements of the robot.

A portable battery will be attached to the robot, which will be connected to the power connection on the Raspberry Pi. This ensures that the robot will not be tethered to an electrical socket as it roams throughout the environment. The Raspberry Pi will be attached to an Arduino Uno via a USB-B cable which will supply power and a data connection to the Arduino Uno, which will provide a two-way line of communication between the devices. Four AA batteries will also be attached to the robot to provide power to the two motors that will drive the robot.
